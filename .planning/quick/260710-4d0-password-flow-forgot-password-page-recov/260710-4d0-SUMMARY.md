# Phase quick-260710-4d0 Plan 01: Password Flow (Forgot / Reset / Change) Summary

Complete Supabase-auth password flow: neutral-state forgot-password request,
`/auth/confirm` recovery-token redemption via `verifyOtp`, a `/reset-password`
page for setting a new password on the recovery session, and an in-app sidebar
"Change password" dialog ‚Äî all reusing one shared new-password form.

## What Was Built

**Task 1 ‚Äî Actions + recovery route + middleware (commit adbf331)**
- `requestPasswordReset(_prev, formData)`: reads `email`, builds server-derived
  `redirectTo = ${siteUrl()}/auth/confirm`, calls `resetPasswordForEmail` inside
  try/catch, and ALWAYS returns `{ sent: true }` (no existence leak).
- `updatePassword(_prev, formData)`: validates `password.length >= 6` and
  `password === confirm`, calls `supabase.auth.updateUser({ password })`,
  `revalidatePath("/", "layout")`, returns `{ success: true }` ‚Äî never redirects.
- `AuthResult` widened with optional `sent?` / `success?` flags (back-compatible).
- `app/auth/confirm/route.ts`: `GET` reads `token_hash` + `type` (typed
  `EmailOtpType`), guards missing params ‚Üí `/login?error=invalid-link`, calls
  `verifyOtp` on the cookie-bound server client, forwards to `/reset-password` on
  success or `/login?error=recovery-failed` on failure. `dynamic = "force-dynamic"`.
- `middleware.ts`: added `/forgot-password` + `/reset-password` to `PUBLIC_PATHS`;
  added `if (pathname === "/reset-password") return supabaseResponse;` before the
  profile-status/approval gate.

**Task 2 ‚Äî Public pages + shared form + login link (commit e554346)**
- `ForgotPasswordForm`: mirrors auth-form card chrome exactly; `useActionState`
  + `useFormStatus`; on `state.sent` swaps the form for a neutral success block
  ("If an account exists for that email, we sent a reset link."); "Back to sign
  in" link.
- `UpdatePasswordForm`: shared form with `{ redirectTo?, onSuccess?, submitLabel? }`;
  two password inputs (`password` + `confirm`, `minLength={6}`); on `state.success`
  a `useEffect` either `router.replace(redirectTo)` or calls `onSuccess?.()` and
  shows an inline "Password updated." line.
- `forgot-password/page.tsx` (one-liner RSC) and `reset-password/page.tsx` (RSC
  that guards the recovery session via `getUser()` ‚Üí `redirect("/login?error=recovery-expired")`
  if absent, then renders the shared form with `redirectTo="/dashboard"`).
- `auth-form.tsx`: login-only right-aligned `text-xs` "Forgot password?" link
  under the password field.

**Task 3 ‚Äî In-app Change password dialog (commit 05d14f5)**
- `ChangePasswordDialog`: `useState` open control; full-width ghost trigger
  (`KeyRound` icon, matches "Sign out" styling); renders the shared
  `UpdatePasswordForm` with `onSuccess={() => setOpen(false)}` (no redirect).
- `sidebar.tsx`: `<ChangePasswordDialog />` inserted between the email line and
  the sign-out form; sign-out form unchanged.
