  <action>
    Edit `isTrustedSource`: immediately BEFORE the final `/[a-z0-9-]+\.[a-z]{2,}/` regex return, add a
    prefix short-circuit `if (s.startsWith("pdf:") || s.startsWith("url:")) return true;` (s is already
    trimmed+lowercased). This honors the existing doc comment ("document sources (pdf:deck.pdf) ...
    count as primary-verified"). Leave the generic/manual/aggregate/unverified/SEC guards above it unchanged.

    Add the upward-outlier guard. Introduce a named constant near the other tolerances:
    `const SPIKE_MULTIPLE = 20; // a legitimate round is never 20x the last known valuation — an untrusted
    figure above this is an outlier/parse-leak, not growth`. Extend the SHARED `rejectionReasons` so it can
    compare against the maximum of the OTHER entries: add a third parameter `otherRefs: { post_money: number }[]`
    (all OTHER dated entries, any source). Inside the existing `if (!isTrustedSource(entry.source))` block,
    after the current down/dup checks, compute the comparison pool as trusted-preferred: `const pool =
    trustedRefs.length ? trustedRefs : otherRefs;` then `const maxOther = pool.reduce((m, o) =>
    Math.max(m, o.post_money), 0);` and push a reason (e.g. "implausible spike — exceeds 20x the highest
    known valuation") when `maxOther > 0 && v > SPIKE_MULTIPLE * maxOther`. When the pool is empty (maxOther===0),
    push nothing — never fabricate a rejection with nothing to compare against. The spike check stays inside the
    untrusted branch, so trusted entries are exempt (trust wins).

    Wire `otherRefs` in BOTH call sites so the guard runs at write time AND in the sweep:
    - In `validateTimeline`: build `otherRefs` for each entry as every OTHER dated entry's post_money
      (exclude the current entry by index/identity so a value never compares against itself); pass it as the
      third arg to `rejectionReasons`.
    - In `filterIngestValuations`: pass all `existing` dated entries' post_money as `otherRefs` (candidates
      are compared against the established set, trusted-preferred via the pool logic above).

    Add tests to the co-located test file: the exact Accrete regression (three entries above → $852B stripped;
    assert isTrustedSource on "pdf:Deal_Overview_-_Accrete_.pdf" is true), the write-time Accrete rejection via
    filterIngestValuations, the no-other-entries survival case, and the trusted-high-entry-exempt case. Do NOT
    weaken or delete existing assertions.
