Structure:
- Private `aktaGet(path, params)`: fetch the base+path with the x-api-key header, zod-parse the envelope `{ data, credits_consumed }`; on ANY failure (no key, network, non-2xx, zod fail) return `null` — never throw (graceful-degradation convention; mirror exa's try/catch + console.error).

Use the returned uuid/website for subsequent calls. When empty → null, so all fetchers return empty sentinels.
- `fetchCompanyProfile(query)` → `GET /v1/company/enrichment?company=<uuid|website>&sections=firmographic`; map via `mapAktaProfile`.
- `fetchNews(query)` → `GET /v1/news?company=<uuid>&limit=15&group_articles=true`; map via `mapAktaNews`. akta returns AI summary + sentiment + publisher metadata natively — no Grok pass.
- `fetchValuationMetric(query)` → `GET /v1/company/enrichment?company=<uuid|website>&sections=financial_estimate`; map via `mapAktaFinancial`. Basis strings MUST say estimate (data-integrity constraint D — never present modeled financials as fact).
- `fetchFundingRounds` → return `[]` with a comment: funding_detail is Enterprise-only and silently absent on lower tiers; rounds keep coming from Grok/Exa/SEC.
