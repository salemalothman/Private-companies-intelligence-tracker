# Phase quick-260717-7g6 Plan 01: Tighten akta Competitor Mention Relevance Summary

Layered a pure, unit-testable precision filter over `AktaConnector.fetchCompetitors` that
drops incidental (<2 mentions), unverifiable, dead, and category-mismatched candidates —
adding NO billable calls (still exactly one `/v1/news` call; free `/v1/company/search`
resolves capped at 10).

## What Was Built

**Task 1 (TDD) — pure relevance functions in `lib/connectors/akta.ts`:**
- `rankIndustryMentions(articles, targetName)`: extracted the mention-counting core from
  `extractIndustryMentions` (three field shapes, case-insensitive target exclusion, unnamed
  dropped), returning `{ name, count }` ranked most-mentioned first with stable first-seen
  tie-break. `extractIndustryMentions` now delegates to it and maps to `ConnectorCompetitor`
  with the byte-for-byte unchanged `basis`/`source` — all pre-existing extractor tests stay
  green.
- `filterRelevantMentions(resolved, targetCategory, opts?)`: four gates — min mentions
  (default 2), verifiability (drop empty `product_category`), liveness (drop
  acquired/delisted/closed/defunct/dead, lenient otherwise), and category similarity
  (shared substantive token OR token-Jaccard >= 0.3). Survivors re-sorted by count desc,
  tagged with the akta basis/source. Pure `categoryTokens` tokenizer (stopword-filtered) and
  `tokenJaccard` helpers back it.

**Task 2 — orchestration wiring in `fetchCompetitors`:**
Rank mentions → gate `count >= 2` and `slice(0, 10)` BEFORE any resolve → per-candidate
best-effort `resolveAktaCompany` under `Promise.all` (miss/failure → null → dropped) →
assemble `resolved` rows → `filterRelevantMentions(resolved, targetCategory)` where
`targetCategory = (company.product_category ?? hint ?? resolvedName).trim()`. Added
`RELEVANCE_RESOLVE_CAP = 10`.
