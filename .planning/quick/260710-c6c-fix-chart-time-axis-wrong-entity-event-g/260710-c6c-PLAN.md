    - A `valuation` event whose title/detail is a generic multi-company sector
      report (title does not contain the tracked company name AND matches
      report-ish keywords like "valuations", "Q2 2026", "sector", "market map",
      "top N", "landscape") is dropped.
    - A legitimate company-specific event (title references the tracked company,
      no wrong-entity signal) passes unchanged with its value intact.
    - When the stored company row exposes profile facts (country / founded_year),
      an event whose text asserts a clearly contradictory foreign exchange origin
      is dropped; absent those facts, the heuristic still works from the
      ticker/domain signals alone.
  </behavior>
  <action>
Extend `lib/enrichment/disambiguation.ts` (mirror its existing style —
observational regex heuristics, pure functions, NO LLM calls, no risk scoring)
with a new exported pure function:

`screenCompanyEvent(company, event)` where
`company: { name: string; country?: string | null; founded_year?: number | null; isPrivate?: boolean }`
and `event: { type: "corporate" | "valuation" | "secondary"; title: string; detail?: string | null; url?: string | null; value?: number | null }`,
returning `{ drop: boolean; value: number | null; reason?: string }`.

Logic (compose existing helpers where possible):
1. Build the combined signal text from title + detail + url (reuse the same
   join approach as the module's private `signalText`).
2. Wrong-entity: if `wrongEntitySignal(company.name, text, { isPrivate: company.isPrivate }).blocked`
   -> `{ drop: true, value: null, reason }`. Strengthen the underlying signals
   so exchange-symbol title patterns (`NYSE:`/`NASDAQ:` in addition to the
   existing `TSE:`/`TYO:`/`\d{4}.T`) and finance/ticker domains
   (`tradingview.com`, `finance.yahoo.`, `marketscreener.com`) are covered —
   extend the module-level `STOCK_SIGNAL` (or add a sibling regex) rather than
   duplicating logic. Keep the "private company only" gating.
3. Profile contradiction (only when facts are present): if `company.country` is
   set and the text asserts a different exchange/HQ country (e.g. Tokyo/Japan via
   `TSE:`/`TYO:` when the company is US), or a stated founded/established year
   differs materially from `company.founded_year`, drop with a reason. Keep this
   conservative — no drop when the facts are absent or ambiguous.
4. Generic multi-company report: add a small pure helper (e.g.
   `isGenericMultiCompanyReport(name, title, detail)`) — true when the title does
   NOT contain the tracked company name AND matches report-ish keywords
   (`valuations?`, `sector`, `market map`, `landscape`, `state of`, `top \d+`,
   `ranking`, `q[1-4]\s?20\d{2}`). For `type === "valuation"` events matching
   this, return `{ drop: true, ... }` (a figure-less valuation event is noise, so
   dropping is cleaner than storing a stripped value).
5. Default: `{ drop: false, value: event.value ?? null }`.

Add co-located tests in `lib/enrichment/disambiguation.test.ts` covering every
bullet in the behavior block, including the two exact live cases: the

value stored) — and at least one positive case that must pass through intact.

Wire into `lib/agents/exa-events.ts`: add `country, founded_year` to the
companies `select`, import `screenCompanyEvent`, and apply it while building
`rows`. Determine `isPrivate` from available data (treat tracked portfolio
companies as private by default — pass `isPrivate: true` unless a public marker
exists). Drop rows the guard rejects; use the guard's returned `value` for the
row's `value` field. Preserve the existing dedupe/pre-filter and best-effort
error handling.
