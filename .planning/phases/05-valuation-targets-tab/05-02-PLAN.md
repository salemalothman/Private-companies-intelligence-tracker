<objective>
Extend the deep-dive agent with a small factual-financials section
(`historical_financials`) so the existing Valuation tab can show
margins/burn/runway/ACV as honestly-labelled qualitative fields (VAL-01, spec §8).
These are `LabelledField`s ONLY — directional text + basis/confidence/source — NOT
a fabricated numeric P&L. This is the Phase 2/3 pattern applied once more: extend
`OverviewSections` + prompt + zod-side shape + `normalizeSections`, add unit
tests, and teach the regen script to report whether the section landed.

Purpose: Give the Valuation tab (Wave 2) real, guardrail-safe financial detail to
render. Output: extended types + agent + tests + regen script.

CRITICAL GUARDRAIL: the da325a4 hardening in runDeepDive — the `MAX_ATTEMPTS`
retry loop and the "bail BEFORE the upsert when sections normalize to zero" no-
clobber guard — MUST be preserved byte-for-byte. This plan only adds one section;
it changes nothing in the retry/upsert control flow.
</objective>
