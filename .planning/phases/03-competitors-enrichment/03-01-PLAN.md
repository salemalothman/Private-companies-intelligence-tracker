<objective>
Extend the deep-dive agent (data side) so a run emits a typed `competitors`
analysis block that CLASSIFIES the already-discovered competitors — it does NOT
re-discover them. The block carries per-competitor threat tiers (CMP-01) and a
Capability Matrix of the target vs. its top 3 threats scored 1-10 (CMP-02).

Purpose: Phase 3 rendering (plan 03-02) needs a stored, normalized `competitors`
section to join onto the existing ranking rows. This mirrors the Phase 2 pattern
(extend types + prompt + zod + normalizeSections + unit tests) exactly.

Output: extended `deep-dive-types.ts`, extended prompt/zod/normalizeSections in
`deep-dive.ts` (with the ranked names threaded in as the classification
allow-list), and unit tests covering the new normalize logic.
</objective>
