<objective>
Build the single Grok deep-dive agent: gather the grounding context the app
already holds, make one structured Grok call for the narrative `sections`, compute
the comps inputs (peer-multiple percentiles + base revenue) deterministically in
code, take ONLY the growth proposal from the LLM, and upsert one `company_analysis`
row. This is the generation engine every downstream tab reads from.

Purpose: FND-02 (one grounded structured agent) + FND-03 (comps math in code, LLM
proposes only growth). Guardrail-critical: no code path fabricates financials or
invents probabilities.
Output: `lib/agents/deep-dive.ts` (`runDeepDive`) + unit tests for the pure comps math.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| web-derived LLM text → stored JSONB → UI | Untrusted Grok output flows into sections/valuation then downstream renders |
| agent → xAI | XAI_API_KEY must stay server-side |
| agent → DB | Writes only the current owner's company_analysis row |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering (data integrity) | comps math | mitigate | Percentiles + base revenue computed in code from SEC-verified peers only; LLM numbers confined to `growth`; no code path fabricates financials or probabilities (enforced by grep gate + unit tests + prompt guard). |
| T-01-05 | Injection | Grok output → SQL | mitigate | Values written via the parameterized supabase-js client (`.upsert(obj)`), never string-concatenated SQL; zod `safeParse` gates the parsed JSON. |
| T-01-06 | Information Disclosure | XAI_API_KEY | mitigate | `import "server-only";` first line; agent runs only in server actions/cron — key never reaches the client bundle. |
| T-01-07 | Spoofing/EoP | wrong-owner writes | mitigate | Under the RLS user client, upsert's `user_id` defaults to `auth.uid()` and the RLS check blocks writes to another owner's company row; the admin client is not used here. |
| T-01-SC | Tampering | npm installs | accept | No new packages — reuses installed `ai`, `@ai-sdk/xai`, `zod`, `@supabase/supabase-js`; nothing to audit. |
