RLS pattern for a per-company table (from supabase/migrations/0006_competitors.sql):
- `user_id uuid not null default auth.uid() references auth.users (id) on delete cascade`
- `company_id uuid not null references public.companies (id) on delete cascade`
- `alter table ... enable row level security;`
- `create policy <name>_all_own for all using (auth.uid() = user_id) with check (auth.uid() = user_id);`
Note: the service-role admin client (createAdminClient) bypasses RLS by design; the
agent writes with the RLS user client from the server action, so user_id defaults to auth.uid().

lib/types.ts conventions:
- Standalone `*Row` / `*Insert` interface pairs (NO nested Database[...] lookups).
- Each table registered in `Database.public.Tables` with Row/Insert/Update/Relationships.
- snake_case column names preserved exactly.

getCompany reader pattern (lib/queries.ts:140) uses `await createClient()` then
`.from(...).eq('id', id).maybeSingle()`, logs `error.message` and returns null on error.

valuation JSONB shape (design spec §4):

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → RLS user client | An authenticated user may only ever touch rows for companies they own |
| server-only admin client → DB | Service-role bypasses RLS by design; must only write rows scoped to the correct owner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | company_analysis RLS | mitigate | Policy `company_analysis_all_own` gates all ops on `auth.uid() = user_id`; user_id defaults to `auth.uid()` on insert so a user cannot write another user's row via the cookie-bound client. |
| T-01-02 | Information Disclosure | cross-user reads | mitigate | Same RLS policy restricts SELECT to the owner; getCompanyAnalysis uses the RLS user client (createClient), not admin. |
| T-01-03 | Tampering | migration integrity | mitigate | [BLOCKING] checkpoint applies and verifies the exact migration before any downstream plan runs against it. |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No new package-manager installs in this plan; nothing to audit. |
</threat_model>
