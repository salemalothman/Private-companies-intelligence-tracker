CLI commands (exact):
- `sec-edgar-pp-cli companies lookup <TICKER>` + AGENT_FLAGS → CIK for a public peer.
- `sec-edgar-pp-cli facts statement --cik <CIK> --kind income --periods last4` + AGENT_FLAGS → XBRL income facts (revenue / net income / etc. by period).
- `sec-edgar-pp-cli cross-section --tag us-gaap:Revenues --ticker A,B,... --periods last8` + AGENT_FLAGS → peer-revenue pivot.
- Free source; requires SEC_EDGAR_USER_AGENT (a contact email) in env — without it the CLI 403s (confirmed). Preflight and skip cleanly if unset.
- PUBLIC peers only have XBRL. Private portfolio targets → no facts → skip (expected, not an error).

PeerFinancialInsert shape (Plan 01): { cik, ticker?, entity_name?, fiscal_period, revenue?, net_income?, gross_profit?, operating_income?, currency?, source?, source_url?, fetched_at? }. Natural key: (cik, fiscal_period). No user_id (shared reference table).

|----------|-------------|
| sec-edgar stdout → cache | untrusted external XBRL JSON, parsed via parseEnvelope, mapped defensively |
