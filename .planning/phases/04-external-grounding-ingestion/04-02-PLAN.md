<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pp-cli stdout → parser | external CLI output is untrusted; must be parsed defensively (parseEnvelope never throws) |
| DB values → CLI args | domain/ticker/subject/cik come from DB and flow into CLI invocation |
| env → script | secrets (X_BEARER_TOKEN, COMPANY_PP_CONTACT_EMAIL, GITHUB_TOKEN) enter only via process.env |
| service-role client → all tables | RLS bypassed; writer must scope company_id/user_id correctly (enforced in Plans 03-05) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering | DB value → CLI arg | mitigate | execFile with array args (no shell) — command injection impossible even for adversarial domain/name values |
| T-04-05 | Tampering | pp-cli stdout parse | mitigate | parseEnvelope validates object shape, never eval/Function, never throws; unit-tested on malformed input |
| T-04-06 | Information Disclosure | secret handling | mitigate | requireEnv reads env only; no secret value is ever logged; missing secret → clean skip, not error dump |
| T-04-07 | Spoofing/fabrication | ambiguous CIK | mitigate | resolveCik returns ambiguous+candidates; caller re-queries by cik and never trusts a name-fragment amount (unit-tested) |
| T-04-SC | Tampering | npm installs | mitigate | no new runtime packages (tsx/vitest already present); slopcheck N/A |
</threat_model>
